Variables are containers that hold information. In Ruby they get represented as a single, connected word that starts with a lowercase letter. We can store information in variables with the = sign.

number = 15

The code snippet above saves the value 15 into the variable number. Now we can use the variable number any place we would have used an actual number. For example, we can now do things like:

puts number + 3

and we get 18 as the result. This only works because we assigned 15 to the variable number

If we want to create a variable in a Rails controller and send it down to the view, we need to put an @ sign at the beginning of the variable. This makes it an instance variable that can move down to the view from the controller:

@number = 15

We can use if-else statements to have our program make decisions. For example if we wanted the program to print "Yay!" if a variable contains "Cubs" and "Booooo" if a variable contains "Sox":

team = "Cubs"
if team == "Cubs"
  puts "Yay!"
else
  puts "Booooo"
end

There are a few things we need to talk about here. The first line just assigns the string "Cubs" into the variable team. The second line does two things. It first uses the == sign to check if team is the same as "Cubs". Notice that we're using the double-equals sign here, not the single-equals sign. Since the variable team contains the string "Cubs", we know that team is the same as "Cubs". Based on whether this statement is true or false, the code either runs the third line or the fifth line.
