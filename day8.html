Today we worked on creating virtual pets using counters and query-strings. Query-string are a way to get information into the params hash without using forms. Here's an example URL with a query string from today:

/process_action?act=feed

The query string starts at the ?. It contains pieces of information called key-value pairs This above query string has one key-value pair, act=feed. The key is act and the value is feed. It's really similar to how the name attribute works in the input element of a form. The name attribute is the key and whatever you type in to the input field is the value. In this case, we're not sending the information through a form, but embedding it directly at the end of a link.

We could also add multiple key-value pairs to a query string by separating them with & symbols:

/process_action?act=feed&amount=3

Now, both of these items will placed in the params hash. We can access the values with params["feed"] and params["amount"].

These commands configure Git with a placeholder name and email. You don't need to use your actual name or email, if you don't want to. You only need to do this once on your computer.

git config --global user.name "John Doe"
git config --global user.email johndoe@example.com

This command tells Git to start keeping track of your project. You only need to do this once per project.

git init

This command tells Git to add all new changes for the next commit

git add -A

This command tells Git to save all the changes that were added. You have to add a message that describes your changes.

git commit -m "first commit"

To connect your local repo to Github, use the following command. Everything in brackets is a placeholder for your repo's information:

git remote add origin [https link to your Github repo]

To push your repo for the first time, use:

git push -u origin master

To clone an existing repository from Github, find the HTTPS url on the bottom right side of the Github repo and paste it in to terminal with the following command:

git clone [url of the Github repo]
git clone "https://github.com/arjunvenkat/git_practice.git"

This should save the repo from Github to your local folder.

To access the app, start from the folder where you typed the clone command and type in:

cd [folder name of the cloned app]

for example, it might look like:

cd git_practice

then bundle the app in case your Rails environment isn't up to date:

bundle install

If you still have problems running the server after 'bundle install', try:

bundle update

Every time you want to save and push your changes, use the following three commands:

git add -A
git commit -m "a message describing any changes"
git push

If you accidentally forgot to add a commit message, you might find yourself in a weird looking editor that you can't get out of. This editor is called Vim, and you can Google commands to navigate inside here. If you want a quick way to save and exit the file, type in:

:wq

If you want to move forwards and backwards in an application's history, use the git checkout command. Find the full SHA of a commit from Github and paste it in with the following command:

git checkout [SHA for a particular commit]

for example:

git checkout e318538471dfaa50398e3679c778014c59763002

This will temporarily revert the application back to an earlier state. If there is a database error, you might need to use the command rake db:migrate to reset the database. /p>

To go back to your application's current state, use:

git checkout master
